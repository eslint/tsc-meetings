### Updating @eslint/config-array for ESLint v9.x (Minimatch Security Issue)

**TSC Summary:** Following up on the minimatch security vulnerability discussion from the previous meeting, the team needs to patch `@eslint/config-array` for ESLint v9.x users.

**Discussion Points:**
- ESLint v9.39.3 uses `@eslint/config-array` v0.21.1
- To patch the minimatch vulnerability for v9 users, need to release v0.21.2
- The only change would be the minimatch update
- In v0.23.0, config-array switched from minimatch 3.x to 10.x
- Upgrading ESLint v9 to use v0.23.x could introduce incompatibilities and would result in two versions of minimatch being downloaded (since ESLint itself also uses minimatch)
- Best approach is to release `@eslint/config-array@0.21.2` with the upgraded minimatch
- For `@eslint/eslintrc`, ESLint v9 ([v3.3.1](https://github.com/eslint/eslintrc/blob/v3.3.1/package.json#L78)) still uses a version of minimatch flagged by security scanners
- Can bump the `@eslint/eslintrc` dependency in ESLint v9 to the latest version to address this
- The maintenance branch should be kept around until v9.x is EOL in case another patch is needed

**Resolution:** The TSC agreed to:
- Release `@eslint/config-array@0.21.2` with upgraded minimatch
- Create a PR for the config-array maintenance release
- Create a PR for ESLint v9 with updated dependencies
- Bump `@eslint/eslintrc` in ESLint v9 to the latest version
- Keep the maintenance branch until v9.x reaches EOL

**Action Items:**
- @nzakas will create the PR for config-array maintenance release
- @fasttime will approve the maintenance release PR after the meeting
- @fasttime will prepare PRs for ESLint targeting the v9.x-dev branch with updated dependencies
