### [Change Request: Evaluate Codemod to replace config migrator and eslint-transforms](https://github.com/eslint/eslint/issues/20442)

**TSC Summary:** The Codemod work for v8-to-v9 config conversion is now complete, and DMartens believes it is ready for a public beta. The TSC discussed how to move forward with publicizing it (e.g., a blog post, updating the migration guides).

**Discussion Points:**
- @fasttime initially noted that the codemod is already at v1.9.36 on the [Codemod registry](https://app.codemod.com/registry/@eslint/v8-to-v9-config), and thought a public beta would start with a prerelease.
- @nzakas clarified that "beta" is just a marketing term here, and the scope is purely v8 → v9 conversion.
- Since the codemod has already been tested and deployed to the registry, the team agreed it only needs to be publicized.
- @nzakas suggested inviting the Codemod team to write a guest blog post explaining that it's now available and ready to try; @fasttime agreed.

**Resolution:** The TSC agreed to invite the Codemod team to write a guest blog post publicizing the availability of the v8-to-v9 config codemod.

### [feat: support TypeScript syntax in `no-unused-vars`](https://github.com/eslint/eslint/pull/19812)

**TSC Summary:** The PR was flagged for TSC attention. The team revisited concerns about the size of the diff and the impact on a high-traffic rule.

**Discussion Points:**
- @nzakas was worried that 4,700 lines of code was a steep price for adding TypeScript support to such a popular rule, particularly with respect to performance.
- @mdjermanovic noted that the rule file diff is actually ~600 lines; the majority of the 4,000+ lines are tests. His earlier concern about depending on `@typescript-eslint/scope-manager` no longer applies.
- @nzakas felt 600 lines was more reasonable but wanted performance validation before a final decision.

**Resolution:** The TSC agreed to review the code and run performance testing to confirm the extra code doesn't cause a significant slowdown before deciding whether to add TypeScript support to `no-unused-vars`.
